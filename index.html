01
The problem

Your agent forgets everything.

🧠

Context resets

Every new session starts from zero. The model has no idea what happened yesterday.

📜

Logs aren't memory

Dumping raw chat history into the prompt doesn't scale, isn't typed, and can't be searched.

🔌

Bolt-on, brittle

Hosted "memory" features lock you in, hide provenance, and offer no way to forget.

Memory isn't a longer context window. It's a system.

02
What it is

A memory system,
not a string list.

MemAtom distills each conversation turn into structured memory atoms — typed, embedded, and provenance-tracked — then surfaces the right ones into your agent's next prompt.

  • Typed: episode · fact · preference · trait · skill
  • Provenance: every atom knows which messages produced it
  • Recallable: hybrid GraphRAG retrieval across all sessions

Before generating, MemAtom recalls relevant atoms and injects them into context. After replying, it extracts new atoms asynchronously — off the hot path, so memory never adds latency to the response.

04
Fits any agentic system

Two calls. Any agent loop.

import { createMemory } from '@mematom/memory-sdk'

const mem = createMemory({ embedder, extractor, repository, agentId })

// …inside YOUR agent loop, whatever framework:
const memories = await mem.recall(userMessage)        // vector + graph
const reply = await llm(withContext(memories, userMessage))
await mem.capture({ userText: userMessage, assistantText: reply })

No framework assumptions. Works with LangChain, custom loops, multi-agent orchestrators — anything that calls an LLM.

05
Retrieval that doesn't miss

GraphRAG recall

Two paths run together and fuse — so you catch what pure vector search misses.

  • Vector: semantic similarity over a pgvector index
  • Graph: entities in the query → 1–2 hop traversal of a knowledge graph
  • Fused + reranked by vector · graph · recency · pin

▢ Vector path

"that thing I worried about last week" → semantic match by embedding.

◈ Graph path

"everyone on Project Titan" → follow entity edges across atoms, even with zero text overlap.

Every recall is explainable

Atoms are tagged with the path that surfaced them — shown as V / G badges in the live view.

06
No embeddings bill. No key.

Embeddings run locally.

🔒

Zero API key

all-MiniLM-L6-v2 runs in-process via transformers.js. ~80MB, downloaded once.

💸

Zero per-call cost

No metered embedding endpoint. Embed millions of atoms for free, fully offline.

🔧

Swap anytime

Behind an Embedder interface — point it at a hosted model later with one line.

Claude has no embeddings endpoint — that's the only reason a second provider ever enters the picture. Local models remove it entirely.

07
Bring your own everything

Three interfaces. Total control.

Embedder

Local MiniLM by default. Swap for OpenAI, Voyage, or any model.

LlmExtractor

Anthropic tool-use included. Plug in any provider that returns structured output.

MemoryRepository

Postgres + pgvector included. The seam for a CRDT / signed-ledger backend.

Everything external is an interface — which is also why the whole SDK is unit-tested with deterministic fakes (zero network, zero flake).

11
Get going

Running in five minutes.

# 1 · install
npm i @mematom/memory-sdk

# 2 · bring a store (Postgres + pgvector)
docker compose up -d db  &&  npm run db:migrate

# 3 · wire it into your agent
const mem = createMemory({ embedder, extractor, repository, agentId })

That's it. The embedder self-loads on boot; the repository handles the rest.
